Starting a new website from scratch can be overwhelming. You don't want to start with "Hello World". You need a basic skeleton that has some basic "batteries included", like brochure content, an about page, a login page that works, and a user-specific dashboard page.

A site starter package should show off basic HTTP calls - see the random quote below.

Every (ok, almost every) site needs to let a user log in, and ideally do it in a modern way. This demo uses JWTs. Try logging in.
