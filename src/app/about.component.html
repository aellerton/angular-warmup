About

The first thing I always want to see is a site that does the following:

  • Shows "brochure" content at the root including "about" *
  • Shows how a user can log in
  • Shows some kind of dashboard when the user is logged in
  • Let's the user log out, and proves that the dashboard is no longer accessible.
* Note I'm not saying that static brochure content "should" be provided by an angular site. I'm still debating that myself. What I'm saying is that I like to see how that's done in any new framework.

I followed these steps:

  1. Read the angular quickstart
  2. Read the [router guide](https://angular.io/docs/ts/latest/guide/router.html) - important because it covers "guarded" routes
  3. Read medium piece on angular with authentication with a back-end (albeit a mocked out one)
  4. Start with a new ng-cli-generated project
  5. Study the plunkr code again and again, re-reading above links, integrating pieces as I went

At this point the basic concept works, but the "logging in" is entirely mocked up, no back-end at all.
